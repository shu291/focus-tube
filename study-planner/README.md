# 逆算プランナー 🎯

大学受験の参考書を「いつまでに・どれだけ終わらせる」で登録すると、**残量 ÷ 残り勉強日数**で
**1日にやる量と目安時間を逆算**してくれる学習計画アプリ。サボった分は自動で残りの日に上乗せされます。

## 何ができる?

### 🎯 逆算スケジュール
- 参考書ごとに **総量(例: 2000単語)・目標日(締切)・勉強する曜日** を登録
- 「今日やる量」を自動計算 → **残量 ÷ 今日から締切までの勉強日数**
- **サボると自動で再計算**: 昨日やらなかった分は残量に残り、残り日数は減るので、
  今日以降の1日のノルマが自動で増える。逆に前倒しすればノルマは減る
- 休養日(勉強しない曜日)は逆算から除外

### 📅 今日(To-Do)
- 参考書ごとに今日のノルマ(量+目安時間)を表示
- **ちょうどノルマ通りにできたら「✓ できた」をタップするだけ**
- 量や時間が違うときは「✎ 記入」で **やった量・かかった時間・メモ** を細かく記録
- **⏱ ストップウォッチ計測**: 「▶ 計測」を押して勉強、止めると自動で今日の時間に加算
  (アプリを閉じても計測は続きます)
- 今日の合計勉強時間・残り冊数・連続日数をひと目で確認

### ⏳ 「今日から毎日 あと何時間」指標
- 各参考書に **「予定どおり終えるには 今日から毎日 ◯時間◯分」** を常時表示
- **サボった翌日・翌々日は、この必要時間が自動で増える**(残量 ÷ 残り勉強日数 × 1単位あたりの時間)
- 遅れているときは **「↑ サボる前は ◯」** と当初ペースを併記し、増えた分がひと目で分かる
- 「今日」タブのヘッダーには **全参考書の合計**(「予定どおり終えるには今日から毎日 合計◯時間」)も表示

### 📊 進捗・予測
- 各参考書の進捗バー・達成率
- **順調 / 遅れぎみ / 前倒し / 締切超過** の状態表示
- **実績ペースからの完了予測日**(「今のペースだと ◯/◯ 完了・締切より △日遅れ」)
- **バーンダウン**(残量の推移を理想線と比較する折れ線)

### 📈 記録
- 日 / 週 / 月の勉強時間を **教科別の積み上げ棒グラフ** で表示
- **GitHub風の学習ヒートマップ**(過去18週間の勉強量)
- 教科別の合計時間、連続記録(🔥)

### 📷 写真
- 参考書の表紙写真を添付(**長辺720pxのJPEGに自動圧縮**してデータを節約)

### 🐙 GitHub連携 & バックアップ
- **プライベートGistに全データ(写真含む)を同期**。機種変更・複数端末で使えます
  - GitHubの個人アクセストークン(`gist` 権限)を設定するだけ
  - 手動の「アップロード / 取得」＋変更時の「自動同期」に対応
  - トークンはこの端末内(localStorage)にのみ保存され、バックアップJSONには含めません
- JSONファイルへの **書き出し / 読み込み** でもバックアップできます

## 使い方

1. `study-planner/index.html` を開く(ホーム画面に追加すればPWAとしてオフラインでも動作)
2. 右下の「＋」で参考書を登録(書名 / 教科 / 総量 / 単位 / 目標日 / 勉強する曜日)
3. 「今日」タブで、ノルマ通りなら **✓**、違えば **✎ 記入**。時間は計測ボタンでもOK
4. 「記録」タブでグラフとヒートマップ、「計画」タブで各参考書の進捗と完了予測を確認

> 💡 参考書を **何周もする** ときは、総量に「ページ数 × 周回数」を入れると、
> そのぶんも逆算に反映されます。

## 逆算の考え方(実装メモ)

各参考書について、その日の目標は次で決まります。

```
今日のノルマ = ⌈ 今日開始時点の残量 ÷ 今日から締切までの勉強日数 ⌉
```

- 「今日開始時点の残量」= 総量 −(登録時点で終わっていた量 + 昨日までにやった量)
- 「勉強日数」= 締切までのうち、勉強する曜日として選んだ日の数(今日を含む)
- 各日ごとに **その日の残量と残り日数から計算し直す** ので、サボり分は自動的に翌日以降へ上乗せされ、
  前倒し分は自動的にノルマが軽くなります
- 目安時間 = 今日のノルマ × 1単位あたりの分。1単位あたりの分は
  **記録がたまると実績(合計分 ÷ 合計量)から自動更新**、なければ登録時の手入力値を使用

## 技術メモ

- 自己完結の単一 `index.html`(ビルド不要・依存なし)
- 計画・記録・設定は `localStorage`(キー: `studyPlanner.v1`)、表紙写真は `IndexedDB`(`study-planner`)に保存
- GitHub同期は `api.github.com` の Gist API を直接呼び出し(CORS対応)
- グラフ・ヒートマップ・バーンダウンはすべて外部ライブラリ非依存(CSS / inline SVG)

## 自動アップデートのしくみ

- このリポジトリは GitHub Actions による **GitHub Pages への自動デプロイ**が設定済み
  (`.github/workflows/pages.yml`)。`main` にマージされると自動でサイトが更新されます
- Service Worker が HTML を **ネットワーク優先** で取得するため、次に開いたときに自動で最新版になります
- 記録データはキャッシュとは別に保存されているので、アップデートで消えません
