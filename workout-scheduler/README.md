# 🏋️ 筋トレ割り当てルーレット

「きょう何の部位をやろう？」「今週どう組もう？」を、**部位ごとの回復時間を考えて自動で割り当てる**アプリです。
ボタンを押すだけで、その日・その週のメニューがランダムに決まります。
やった部位・種目は**カレンダー**で一覧でき、合計回数も集計されます。

## できること

- **きょう**タブ … 各部位の「回復ぐあい」を見ながら、いちばん休めている部位を軸に今日のメニューをランダム割り当て。引き直しも可。
- **1週間**タブ … 週3〜6日を選ぶと、同じ部位が連続しないよう休養を空けた1週間のスケジュールを自動生成。カレンダーに反映できます。
- **カレンダー**タブ … 実施した部位を色ドットで一覧。予定は点線枠で表示。タップで種目の詳細、削除、予定→実施の切替。合計回数も表示。
- 🔥 連続記録（ストリーク）バッジ
- 記録はこの端末のブラウザ（localStorage）に保存。外部送信はありません。JSON書き出しも可能。

## 科学的根拠(回復時間)について

筋肥大では「1部位あたり**中2〜3日（約48〜72時間）**空けて、**週2回前後**の頻度」が一般的に推奨されます。
本アプリは部位ごとに目安の回復時間を設定し、回復が終わった部位を優先して割り当てます。

| 部位 | 目安の回復時間 |
|---|---|
| 脚 | 約72時間 |
| 胸 / 背中 | 約48時間 |
| 肩 / 腕 | 約48時間 |
| 腹 | 約24時間 |

※あくまで一般的な目安です。体調・睡眠・栄養により個人差があります。

## つかいかた

1. **きょう**タブで「🎲 今日のメニューを割り当てる」→ 気に入ったら「✅ 記録」
2. **1週間**タブで日数を選び「🗓️ 今週のスケジュールを割り当てる」→「📌 カレンダーに反映」
3. **カレンダー**タブでこれまでの記録をふりかえる

## 公開方法 (GitHub Pages)

1. リポジトリの **Settings → Pages** で公開ブランチ(`main` / `(root)`)を選択
2. 公開後、`https://<ユーザー名>.github.io/focus-tube/workout-scheduler/` でアクセスできます

## 構成

- `index.html` … 画面（きょう / 1週間 / カレンダー）
- `style.css` … 暗めのデザイン
- `app.js` … 割り当てロジック・記録・カレンダー・統計
- `manifest.webmanifest` / `sw.js` … PWA（ホーム追加・オフライン起動）

外部ライブラリ不要・ビルド不要のバニラ構成です。GitHub と連携しているので、何度でも作り直せます。
